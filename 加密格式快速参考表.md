# AnimeStudio 加密格式快速参考

## 📋 BLK 格式速查表

| 项目 | 值 | 说明 |
|------|-----|------|
| **Data Offset** | `0x2A` (42) | 数据开始位置 |
| **Key Size** | `0x1000` (4096) | 固定密钥大小 |
| **Seed Block Size** | `0x800` (2048) | 基础种子大小 |
| **XOR Pad Size** | `0x1000` (4096) | XOR垫大小 |
| **头部结构** | Sig+Key+Seed | null分割+大小字段 |

## 🔐 BLK 解密步骤

```
1. 解析头部 ← 获取 Signature, Key大小, Key数据, Seed大小
2. 处理Key
   2.1 应用SBox (可选)
   2.2 AES-ECB解密(Key, ExpansionKey)
   2.3 XOR InitVector
3. 计算Seed = KeyLow ⊕ KeyHigh ⊕ XOR(SeedData) ⊕ InitSeed
4. 生成XOR Pad ← MT19937_64(Seed) × 512次
5. XOR解密 ← 从0x2A开始循环XOR
```

## 📋 BLB3 格式速查表

| 项目 | 值 | 说明 |
|------|-----|------|
| **Header Size** | 16 | 加密头大小 |
| **Max Buffer** | 128 | 最大处理缓冲区 |
| **RC4 Key Size** | 256 | 标准RC4 |
| **GF256矩阵** | 各种 | 有限域运算 |

## 🔐 BLB3 解密步骤

```
1. 提取前16字节作为加密头
2. 初始XOR ← buffer[0:16] ⊕ header
3. AES自定义加密 ← BlbAES.Encrypt(buffer[0:16], header)
4. RC4变体 ← 如果length > 16
   4.1 KSA (Key Scheduling Algorithm)
   4.2 PRGA (Pseudo-Random Generation Algorithm)
5. Descramble (3轮)
   5.1 ShiftRow索引
   5.2 GF256乘法
   5.3 SBox替换
```

## 🛠️ Python 实现快速开始

### BLK 解密

```python
from blk_decryptor import BLKDecryptor, GameParams

params = GameParams(
    name="GenshinImpact",
    init_vector=bytes.fromhex('E3FC2D269CC5A2ECD3F8C6D377C249B9'),
    expansion_key=bytes.fromhex('542FED67...'),
    init_seed=0x567BA22BABB08098,
)

decryptor = BLKDecryptor(params)
with open('video.blk', 'rb') as f:
    encrypted = f.read()

decrypted = decryptor.decrypt(encrypted)
with open('video_dec.ivf', 'wb') as f:
    f.write(decrypted)
```

### 格式检测

```python
def detect_format(data: bytes) -> str:
    """自动检测 BLK vs BLB3"""
    try:
        null_pos = data.index(b'\x00')
        if 0 < null_pos < 64:
            # 可能是BLK
            sig = data[:null_pos]
            try:
                sig.decode('utf-8')
                return 'BLK'
            except:
                pass
    except:
        pass
    return 'BLB3'
```

## 📊 算法复杂度对比

### 时间复杂度

| 操作 | BLK | BLB3 |
|------|-----|------|
| MT19937初始化 | O(312) | - |
| XOR Pad生成 | O(512) | - |
| 数据XOR | O(n) | O(128) |
| RC4 PRGA | - | O(112) |
| Descramble | - | O(48) |
| **总体** | **O(n+512)** | **O(128+48+112)** |

### 空间复杂度

| 结构 | BLK | BLB3 |
|------|-----|------|
| MT状态 | 2,500 B | - |
| XOR Pad | 4,096 B | - |
| RC4 S-box | - | 256 B |
| 临时缓冲区 | 4,096 B | 128 B |
| **总体** | **>10 KB** | **<1 KB** |

## ⚠️ 常见陷阱

| 陷阱 | BLK | BLB3 |
|------|-----|------|
| **字节顺序** | 小端序 | 小端序 |
| **Key大小验证** | ✓ 必须 | ✗ 固定 |
| **溢出处理** | 64位掩码 | 8位掩码 |
| **SBox应用** | 有条件 | 总是 |
| **AES密钥方向** | 解密 | 加密(特殊) |

## 🔍 调试检查清单

### 文件头验证
- [ ] Signature可解码为UTF-8
- [ ] Key大小在1-16384范围
- [ ] Seed大小在0-4096范围
- [ ] Header总大小合理

### Key处理验证
- [ ] SBox应用(若需要)前后的Key变化
- [ ] AES输出不是全零
- [ ] XOR后Key与原始不同

### Seed计算验证
- [ ] KeyLow, KeyHigh分别读取
- [ ] KeySeed是所有seed数据的XOR
- [ ] 最终Seed是64位无符号整数

### XOR Pad验证
- [ ] Pad大小恰好4096字节
- [ ] Pad的第一个8字节不全为0
- [ ] 解密时Pad循环索引正确
